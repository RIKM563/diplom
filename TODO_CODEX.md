## Код: будущая задача по BayesianRiskEventAdjuster

Байесовский модуль пока не реализовывать без отдельной команды.

Когда придёт время, сделать модуль:

risk_system/event_pipeline/bayesian_adjuster.py

Название класса:

BayesianRiskEventAdjuster

Место в pipeline:

risk_events = RiskEventClassifier.classify_many(...)
risk_events = BayesianRiskEventAdjuster.adjust_many(risk_events)
assessments = RiskEventAssessmentEngine.assess_many(risk_events)

Назначение:
- уточнять probability_estimate;
- учитывать зависимости между threat_scenario;
- писать evidence в metadata/rationale;
- не считать риск самостоятельно.

Пример зависимости:

phishing → unauthorized_access
unauthorized_access → privilege_escalation
privilege_escalation → data_leak
misconfiguration → malware

Формула допустима простая:

p_new = p_old + p_parent * influence * (1 - p_old)

с ограничением p_new <= 1.

### Запрещённые элементы

Не использовать:

- risk = probability × impact × criticality;
- отдельный множитель criticality;
- риск-балл;
- опасное состояние;
- истинная вероятность угрозы;
- совокупный риск банка as normative indicator;
- ограничение по сроку внедрения T.

### Обязательные элементы

Использовать:

r_ij^0 = p_ij^* · I_ij

r_ij(x) = max(0, r_ij^0 - sum_h a_ijh x_h)

sum_h C_h x_h <= B

sum_h L_h x_h <= L_max

x_h ∈ {0,1}

R(x) = sum_i sum_j r_ij(x) → min

При необходимости:

sum_h x_h <= K
